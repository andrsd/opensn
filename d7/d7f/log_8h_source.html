14/**Object for controlling logging.
15 *
16 * ## Part A: Output logs
17 * There are three levels of verbosity in openSn: Zero(Default), One and Two.
18 * These can be set on the command line via the switch -v followed by a
19 * space and the number for the verbosity (0,1 or 2).
20 *
21 * \code
22 * ./opensn InputFile.lua -v 1
23 * \endcode
24 *
25 * The lua command `LogSetVerbosity(int_level)` achieves the same.\n\n
26 *
27 * Printing a log under the auspices of a verbosity level again has
28 * numerous options. Firstly, any log can be a normal log, a warning
29 * or an error. Secondly, a log can be either location 0 or all the locations
30 * in a parallel process environment. The log option enums defined under
31 * LOG_LVL are
32 - LOG_0, Used only for location 0
33 - LOG_0WARNING, Warning only for location 0
34 - LOG_0ERROR, Error only for location 0
35 - LOG_0VERBOSE_0, Default verbosity level
36 - LOG_0VERBOSE_1, Used only if verbosity level equals 1
37 - LOG_0VERBOSE_2, Used only if verbosity level equals 2
38 - LOG_ALL, Verbose level 0 all locations
39 - LOG_ALLWARNING, Warning for any location
40 - LOG_ALLERROR, Error for any location
41 - LOG_ALLVERBOSE_0, Default verbosity level
42 - LOG_ALLVERBOSE_1, Used only if verbosity level equals 1
43 - LOG_ALLVERBOSE_2, Used only if verbosity level equals 2
44
45 * A log can be made by first connecting code with the logger. This is done
46 * by including the log header and then defining an extern reference to the
47 * global object.
48 *
49 * \code
50 * #include "framework/logging/log.h"
51 * extern Logger& opensn::log;
52 * \endcode
53 *
54 * A log is then written inside a piece of code as follows:
55 *
56 * \code
57 * void PrintSomethingToLog()
58 * {
59 * opensn::log.Log() << "This is printed on location 0 only";
60 * opensn::log.Log0Warning() << "This is a warning";
61 * opensn::log.Log0Error() << "This is an error";
62 * }
63 * \endcode
64 *
65 \verbatim
66 [0] This is printed on location 0 only
67 [0] **WARNING** This is a warning
68 [0] **!**ERROR**!** This is an error
69 \endverbatim
70 *
71 * ## Part B: Repeating events log
72 * Suppose a routine or segment of code gets called multiple times. Now
73 suppose
74 * we want to know how many times this routine was called, how long it ran
75 * (total and on-avg), or we want to know at which timestamp it ran every
76 time.
77 * We do this by using a repeating event. A repeating event is identified
78 * by a tag. Therefore the first step is to obtain a unique id for the event
79 using
80 * Logger::GetRepeatingEventTag.
81 *
82 * \code
83 * size_t tag = opensn::log.GetRepeatingEventTag(std::string("Sweep Timing"))
84 * \endcode
85 *
86 * Events can now be logged using
87 * Logger::LogEvent(size_t ev_tag, Logger::EventType ev_type). As an example,
88 * consider we want to count the number of occurrences of an event. We trigger
89 * multiple occurences using
90 *
91 * \code
92 * opensn::log.LogEvent(tag,opensn::Logger::EventType::SINGLE_OCCURRENCE);
93 * \endcode
94 *
95 * At the end of the block for which you wanted to log these events we can now
96 * extract the number of occurrences using
97 *
98 * \code
99 * double number_of_occ =
100 opensn::log.ProcessEvent(tag,opensn::Logger::EventOperation::NUMBER_OF_OCCURRENCES);
101 *\endcode
102 *
103 * Below is a complete example:
104 *
105 \code
106 size_t tag = opensn::log.GetRepeatingEventTag(std::string());
107
108 for (int i=0; i<5; ++i)
109 opensn::log.LogEvent(tag,Logger::EventType::SINGLE_OCCURRENCE);
110
111 opensn::log.LogAll()
112 << opensn::log.ProcessEvent(tag,
113 Logger::EventOperation::NUMBER_OF_OCCURRENCES); \endcode \verbatim
114 6
115 \endverbatim
116
117 * Since opensn::log is a global object this functionality is really powerful for
118 * use in modules or class objects where multiple methods can contribute to
119 * the same event track. For example an object can have an event tag as a member
120 * and initialize it at construction then all of the objects methods can
121 * contribute to the event.
122 *
123 * ### Supplying event information
124 * In addition to the Logger::EventType the user can also supply a reference to
125 * a Logger::EventInfo structure. Be cautious with this though because each
126 * event stored a double and a string which can look like a memory leak if
127 * multiple events are pushed up with event information. Developers can supply
128 * either a double or a string or both to an event info constructor to
129 * instantiate an instance. The event arb_value is by default 0.0 and the event
130 * arb_info is by default an empty string. An example is shown below:
131 *
132 \code
133 size_t tag = opensn::log.GetRepeatingEventTag(std::string());
134
135 for (int i=0; i<5; ++i)
136 {
137 auto ev_info = std::make_shared<opensn::Logger::EventInfo>(i*2.0);
138 opensn::log.LogEvent(tag,opensn::Logger::EventType::SINGLE_OCCURRENCE,ev_info);
139 }
140
141 opensn::log.LogAll()
142 << opensn::log.ProcessEvent(tag, opensn::Logger::EventOperation::AVERAGE_VALUE);
143 \endcode
144 \verbatim
145 4.0
146 \endverbatim
147 *
148 * Event information can also be supplied by string values but then the
149 * average value is meaningless unless the average value is also supplied.
150 * For example:
151 *
152 \code
153 size_t tag = opensn::log.GetRepeatingEventTag(std::string());
154
155 opensn::log.LogEvent(tag,
156 opensn::Logger::EventType::SINGLE_OCCURRENCE,
157 std::make_shared<opensnLogger::EventInfo>(std::string("A"),2.0));
158 opensn::log.LogEvent(tag,
159 opensn::Logger::EventType::SINGLE_OCCURRENCE,
160 std::make_shared<opensn::Logger::EventInfo>(std::string("B")));
161 opensn::log.LogEvent(tag,
162 opensn::Logger::EventType::SINGLE_OCCURRENCE,
163 std::make_shared<opensn::Logger::EventInfo>(std::string("C"),2.0));
164
165 opensn::log.LogAll() << opensn::log.ProcessEvent(tag, Logger::EventOperation::AVERAGE_VALUE);
166 \endcode
167 \verbatim
168 1.33333
169 \endverbatim
170 *
171 * To get a string value of the event history developers can use
172 * Logger::PrintEventHistory along with the event tag. Just note that it will
173 * automatically be formatted for each location so no need to use opensn::log to
174 * print it. Also, each event will be prepended with a program timestamp
175 * in seconds.
176 *
177 \code
178 std::cout << opensn::log.PrintEventHistory(tag);
179 \endcode
180 \verbatim
181 1.33333
182 [0] 3.813119000 EVENT_CREATED
183 [0] 3.813120000 SINGLE_OCCURRENCE A
184 [0] 3.813121000 SINGLE_OCCURRENCE B
185 [0] 3.813122000 SINGLE_OCCURRENCE C
186 \endverbatim
187 * */
